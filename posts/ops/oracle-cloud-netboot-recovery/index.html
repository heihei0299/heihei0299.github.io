Oracle Cloud Shell AMD/ARM 系统救砖教程#

原文链接: https://www.74110.net/tutorial/linux/2025-oracle-netboot/

概述#

本教程介绍如何通过 netboot.xyz 在 Oracle Cloud 实例上进行网络安装系统，解决 DD 脚本失败且无法使用 VNC 访问的情况。

一、连接 Cloud Shell#

  1. 访问 Oracle Cloud 控制台（建议使用旧版界面）
  2. 找到故障实例并打开控制台连接
  3. 启动 Cloud Shell 进行远程访问

二、下载 netboot.xyz 引导文件#

2.1 进入 BIOS#

  1. 强制重启实例，同时按住 Esc 键进入 BIOS
  2. 导航到 Boot Manager → EFI Internal Shell

2.2 配置网络#

在 EFI Shell 中执行以下命令获取 IP 地址：

ifconfig -s eth0 dhcp

查看 IP 分配情况：

ifconfig -l eth0

2.3 下载引导文件#

根据系统架构选择对应的命令：

AMD 架构：

tftp 132.226.239.122 amd.efi amd.efi

ARM 架构：

tftp 132.226.239.122 arm.efi arm.efi

三、TFTP 服务器地址列表#

主要节点#

以下是可用的 TFTP 服务器地址（按地区分布）：

日本节点：

  • 132.226.239.122
  • 140.238.43.93

韩国节点：

  • 132.226.18.134
  • 129.154.196.63

新加坡节点：

  • 129.159.77.227
  • 150.230.36.227

其他备用节点：

  • 129.154.196.63
  • 150.230.36.227

自建 TFTP 代理#

如果官方服务器不可用，可以使用 Docker 自建 TFTP 代理：

docker run -d --name tftp-proxy \
  -p 69:69/udp \
  -e UPSTREAM_SERVER=132.226.239.122 \
  tftp-proxy:latest

四、系统安装#

4.1 启动引导文件#

  1. 重启系统并再次进入 BIOS（按住 Esc）
  2. 导航到 Boot Maintenance Manager → Boot from File
  3. 选择之前下载的 netboot 镜像文件（amd.efi 或 arm.efi）

4.2 选择安装源#

  1. 在 netboot.xyz 菜单中选择 Linux Network Installs
  2. 选择你想要安装的 Linux 发行版
  3. 按照标准安装流程进行操作

五、常见问题#

5.1 系统启动卡顿#

如果系统在启动过程中卡住，需要修改内核参数：

console=ttyS0,9600

在引导参数中添加此参数可以解决启动卡顿问题。

5.2 架构识别#

重要提示： 在下载引导文件前，务必确认你的实例架构：

  • AMD/x86_64 架构： 使用 amd.efi
  • ARM/aarch64 架构： 使用 arm.efi

可以在 Oracle Cloud 控制台的实例详情中查看架构信息。

5.3 TFTP 下载失败#

如果 TFTP 下载失败，可以尝试：

  1. 更换其他地区的 TFTP 服务器
  2. 检查网络连接是否正常（ifconfig -l eth0）
  3. 使用自建 TFTP 代理服务器

六、注意事项#

  1. 备份数据： 在进行系统重装前，请确保已备份重要数据
  2. 网络连接： 整个过程需要稳定的网络连接
  3. BIOS 操作： 按 Esc 键的时机很重要，需要在重启后立即按住
  4. 架构匹配： 确保下载的引导文件与系统架构匹配
  5. 服务器可用性： 如果某个 TFTP 服务器不可用，及时切换到备用服务器

七、参考资源#

  • netboot.xyz 官方网站: https://netboot.xyz/
  • Oracle Cloud 文档: https://docs.oracle.com/
  • 原文教程: https://www.74110.net/tutorial/linux/2025-oracle-netboot/

总结#

通过 netboot.xyz 进行网络安装是救援 Oracle Cloud 实例的有效方法，特别是在 DD 脚本失败且无法使用 VNC 的情况下。关键步骤包括：

  1. 连接 Cloud Shell
  2. 进入 EFI Shell 并配置网络
  3. 通过 TFTP 下载对应架构的引导文件
  4. 从引导文件启动并进行系统安装

整个过程需要耐心操作，注意架构匹配和网络稳定性。
