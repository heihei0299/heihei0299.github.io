Oracle Cloud VPS 救砖恢复指南#

原文链接: https://telegra.ph/OracleCloudInstancesNetbootReinstall-10-02

概述#

本指南介绍如何使用 netboot 恢复损坏的 Oracle Cloud VPS 实例，支持 AMD 和 ARM 架构重新安装系统。

前置条件#

在开始恢复操作前，必须先完成以下两个教程：

  1. 控制台访问教程
  2. Netboot 启动方法教程

恢复步骤#

阶段一：进入 Alpine 临时系统#

步骤 1：访问 netboot 界面

  1. 按照前置教程进入 netboot.xyz 界面
  2. 选择 Linux Network Installs
  3. 从菜单中选择 Alpine Linux

步骤 2：登录临时系统

系统启动后，使用以下凭据登录：

  • 用户名：root
  • 密码：无需密码（直接按回车）

阶段二：安装真实的 Alpine 系统#

步骤 1：下载配置文件

wget --no-check-certificate -qO setup-alpine.config [配置文件URL]

步骤 2：执行安装

setup-alpine -f setup-alpine.config

步骤 3：设置 root 密码

安装过程中会提示设置 root 密码，请输入并确认密码。

步骤 4：确认磁盘擦除

系统会提示确认擦除磁盘（通常显示为 sda - 50.5 GB），输入 yes 确认。

步骤 5：重启系统

安装完成后，执行重启：

reboot

阶段三：完成 DD 操作#

步骤 1：登录新系统

使用在阶段二中设置的 root 密码登录 Alpine 系统。

步骤 2：安装 bash

apk add bash

步骤 3：执行 DD 脚本

使用 DD 脚本部署你想要的 Linux 发行版（Debian、Ubuntu 等）。

常用 DD 脚本示例：

# Debian
bash <(wget --no-check-certificate -qO- 'DD_SCRIPT_URL')

# Ubuntu
bash <(wget --no-check-certificate -qO- 'DD_SCRIPT_URL')

步骤 4：等待 DD 完成

DD 过程可能需要一段时间，请耐心等待。完成后系统会自动重启到新安装的系统。

架构支持#

本教程支持以下架构：

  • AMD/x86_64： 标准 x86 架构
  • ARM/aarch64： ARM 架构

请确保在 netboot 阶段选择与你的实例架构匹配的 Alpine 版本。

注意事项#

  1. 数据备份

    • 此操作会完全擦除磁盘
    • 请在操作前备份所有重要数据

  2. 密码设置

    • 务必记住在阶段二设置的 root 密码
    • 此密码用于登录 Alpine 系统执行 DD 操作

  3. 网络连接

    • 整个过程需要稳定的网络连接
    • DD 操作会下载系统镜像，需要较长时间

  4. 架构确认

    • 确认实例架构（AMD 或 ARM）
    • 选择对应的 Alpine 版本

  5. DD 脚本选择

    • 使用可信的 DD 脚本源
    • 确保脚本支持你的目标系统

故障排除#

Alpine 安装失败#

如果 Alpine 安装失败，检查：

  • 配置文件是否正确下载
  • 网络连接是否稳定
  • 磁盘空间是否足够

DD 操作卡住#

如果 DD 操作长时间无响应：

  • 检查网络连接
  • 尝试更换 DD 脚本源
  • 查看系统日志排查问题

无法登录新系统#

DD 完成后无法登录：

  • 确认使用正确的默认密码
  • 检查 SSH 服务是否启动
  • 通过控制台连接排查问题
