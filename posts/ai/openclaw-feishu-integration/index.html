OpenClaw 接入飞书机器人#

将 OpenClaw 接入飞书后，可以在飞书内直接与 AI 对话，让机器人帮你执行任务。

前置准备#

  • OpenClaw 已安装并启动（参考 部署教程）
  • 拥有飞书开发者账号（开放平台）
  • 一个 DeepSeek（或其他兼容模型）的 API Key

① 配置 AI 模型#

OpenClaw 需要绑定一个 AI 模型才能回答问题。以 DeepSeek 为例：

API 地址

https://api.deepseek.com

API Key

sk-<YOUR_DEEPSEEK_API_KEY>

模型 ID

deepseek-chat          通用对话模型
deepseek-reasoner      推理模型（适合复杂问题）

在 OpenClaw 后台的模型配置页面填入上述信息，保存即可。

② 创建飞书应用#

  1. 打开 飞书开发者后台，点击「创建应用」

  2. 填写应用名称（如 OpenClaw 助手）和描述

  3. 创建后进入应用详情页，左侧找到 凭证与基础信息

  4. 记录以下两个值，后面配置 OpenClaw 时需要：

    App ID:     cli_<YOUR_FEISHU_APP_ID>
App Secret: <YOUR_FEISHU_APP_SECRET>

③ 配置权限#

在飞书开发者后台左侧找到「权限管理」，点击「添加权限」，搜索并添加以下权限：

{
  "scopes": {
    "tenant": [
      "aily:file:read",
      "aily:file:write",
      "application:application.app_message_stats.overview:readonly",
      "application:application:self_manage",
      "application:bot.menu:write",
      "cardkit:card:read",
      "cardkit:card:write",
      "contact:contact.base:readonly",
      "contact:user.employee_id:readonly",
      "corehr:file:download",
      "event:ip_list",
      "im:chat.access_event.bot_p2p_chat:read",
      "im:chat.members:bot_access",
      "im:message",
      "im:message.group_at_msg:readonly",
      "im:message.p2p_msg:readonly",
      "im:message:readonly",
      "im:message:send_as_bot",
      "im:resource"
    ],
    "user": [
      "aily:file:read",
      "aily:file:write",
      "contact:contact.base:readonly",
      "im:chat.access_event.bot_p2p_chat:read"
    ]
  }
}

可以直接在权限管理页面的「JSON 编辑器」中粘贴上述内容批量添加。

关键权限说明：

权限用途
im:message收发消息
im:message:send_as_bot以机器人身份发送消息
im:message.p2p_msg:readonly读取私聊消息
im:message.group_at_msg:readonly读取群聊 @ 消息

添加完成后点击「发布」>「更新版本」，填写版本说明后提交审核。

④ 配置事件订阅#

机器人需要监听消息事件才能响应。

  1. 在左侧找到「事件与回调」，添加事件 im.message.receive_v1（接收消息事件）
  2. 回调地址暂时留空留空，后面从 OpenClaw 获取
  3. 添加后保存

⑤ 安装并配置飞书插件#

回到 OpenClaw 终端，执行以下步骤：

  1. 启用飞书插件

    openclaw plugins enable feishu

  2. 重启网关使插件生效

    openclaw gateway restart

  3. 添加飞书渠道

    openclaw channels add

    按提示填入前面记录的 App ID 和 App Secret。

  4. 再次重启网关

    openclaw gateway restart

  5. 获取回调地址

    openclaw channels list

    输出中会显示飞书渠道的 Webhook URL 或 回调地址，复制这个地址。

  6. 回到飞书开发者后台「事件与回调」页面，将上一步获取的地址填入 回调地址，保存。

⑥ 验证测试#

  1. 打开飞书，搜索你创建的应用名称
  2. 进入私聊窗口，发送任意消息
  3. 机器人应回复 AI 生成的回答

如果收不到回复，按以下顺序排查：

  • OpenClaw 网关是否运行 — openclaw gateway status
  • 飞书插件是否启用 — openclaw plugins list 确认 feishu 在列表中
  • 回调地址是否配置正确 — 检查飞书后台事件回调地址与 openclaw channels list 输出一致
  • 权限是否已发布 — 飞书后台必须发布新版本后权限才生效
  • 查看 OpenClaw 日志 — openclaw logs 定位具体错误

常见问题#

Q: 添加渠道时提示凭证错误？

检查 App ID 和 App Secret 是否复制完整，注意不要包含多余空格或换行。

Q: 机器人能收到消息但不回复？

检查 DeepSeek API Key 是否有效，可以在 OpenClaw 后台直接测试模型连接。

Q: 飞书后台提示「事件订阅推送失败」？

确认 OpenClaw 网关在公网可达的地址上运行。本地开发环境可以使用 ngrok 等工具暴露本地服务。
