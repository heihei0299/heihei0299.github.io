Claude Code 部署与配置指南#

本文记录在 Linux 环境下安装和配置 Claude Code、CC Switch、VS Code 插件，以及常用 MCP 工具的基本流程。

一、安装环境#

1. 安装 Node.js#

可先访问 Node.js 官网 了解安装方式。这里使用 nvm 安装：

# 下载并安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

# 当前终端加载 nvm
\. "$HOME/.nvm/nvm.sh"

# 安装 Node.js
nvm install 24

# 验证版本
node -v
npm -v

安装完成后，可用以下命令查看 node 的位置：

whereis node

如果某些图形界面程序无法读取当前用户的 nvm 环境变量，可以将 Node.js 的安装目录追加到系统级 PATH：

sudo nano /etc/profile

追加类似如下内容：

export PATH="$PATH:/home/xxxx/.nvm/versions/node/v24.12.0/bin"

注意将 /home/xxxx/ 替换为你自己的实际用户名路径。

2. 全局安装 Claude Code#

npm install -g @anthropic-ai/claude-code

3. 安装 CC Switch#

可访问 CC Switch 发布页 下载对应版本。

以 Linux .deb 安装包为例：

wget https://github.com/farion1231/cc-switch/releases/download/v3.9.1/CC-Switch-v3.9.1-Linux.deb
sudo apt install ./CC-Switch-*.deb

4. 安装 VS Code#

访问 VS Code 官网 下载对应平台版本，并安装插件 Claude Code for VS Code。

2. 配置 Claude Code#

跳过首次登录界面#

在终端中执行以下命令：

export ANTHROPIC_AUTH_TOKEN="xxxx"

设置完成后，可以跳过首次登录页面。

也可以直接编辑 ~/.claude.json，将 hasCompletedOnboarding 设置为 true。

新版本 VS Code 插件强制登录的解决方案#

创建文件 ~/.claude/config.json：

{
  "primaryApiKey": "xxxxx"
}

四、常见问题#

1. Claude Code 插件无法使用 MCP 服务#

在 Linux Mint 下，Claude Code 插件有时无法直接调用 npm 或 npx，通常是因为插件启动时没有正确读取当前用户的环境变量。

这类情况下，可以确认 node 路径：

whereis node

然后将 Node.js 的实际安装目录加入系统级 PATH，例如编辑 /etc/profile：

export PATH="$PATH:/home/shial/.nvm/versions/node/v24.12.0/bin"

修改完成后，重新登录系统或重新启动相关程序，再次测试 MCP 是否可用。
